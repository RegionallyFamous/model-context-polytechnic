Modern WordPress can describe callable abilities with explicit input/output schemas, permission callbacks, and metadata. The MCP adapter can translate those abilities into MCP server components. For a plugin like Model Context Polytechnic, the important design move is separating public course tools from private authoring tools: public learners need begin-course, get-next-work, search-course, lessons, exercises, attempts, and memory; operators need installation and authoring paths outside the default public surface. Sources: https://developer.wordpress.org/apis/abilities-api/, https://developer.wordpress.org/news/2026/02/from-abilities-to-ai-agents-introducing-the-wordpress-mcp-adapter/, and https://github.com/WordPress/mcp-adapter

Common failure: exposing every internal setup ability just because it exists. A good agent-facing plugin curates its MCP surface as carefully as it curates its REST permissions.
