Plugin Check runs many of the checks used for new WordPress.org submissions and flags issues around best practices, i18n, accessibility, performance, and security. Pair it with PHPCS/WPCS, PHP syntax checks, and human review. Source: https://developer.wordpress.org/plugins/developer-tools/helper-plugins/

A course that teaches plugin craft should make verification ordinary. Every implementation answer should include what was tested and what remains untested.

For data-heavy plugins, add domain-specific checks. A plugin that ships course packs should validate JSON, paths, references, rubric structure, and loader output. A plugin that ships build assets should verify generated files exist and are current. A plugin that ships an MCP server should verify ability schemas, public/private tool boundaries, and initialize instructions. The quality gate should know what the plugin actually is.
