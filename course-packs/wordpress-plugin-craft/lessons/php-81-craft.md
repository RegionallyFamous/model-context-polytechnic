PHP 8.1+ gives plugin authors better type declarations, readonly properties, enums, first-class callables, and safer object modeling. WordPress APIs still often speak arrays, callbacks, WP_Error, and globals. The craft is to put a clean boundary between modern PHP internals and WordPress-facing integration code. Source: https://www.php.net/manual/en/

A good LLM answer should know when a small procedural callback is clearer than a framework-shaped abstraction.
