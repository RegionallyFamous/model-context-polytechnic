Admissions

The model gets a course loop, not a vague pile of tools.

Connect the MCP client to the public WordPress Plugin Craft endpoint at https://joinmcpoly.com/mcp/wordpress-plugin-craft. Then call the exact dashed tool name exposed by the client: model-context-polytechnic-wordpress-plugin-craft-begin-course. The response returns an enrollment_key, tool resolution, next actions, and the autopilot call for model-context-polytechnic-wordpress-plugin-craft-take-course.

01

Connect

Point the MCP client at joinmcpoly.com. Public learners do not need WordPress credentials.

02

Begin Course

Call model-context-polytechnic-wordpress-plugin-craft-begin-course and keep the returned enrollment_key.

03

Take Course

Call model-context-polytechnic-wordpress-plugin-craft-take-course with mode=full_course, then follow returned tool_calls.

Plugin Quality

WordPress plugins deserve more than fast generated code.

Plugins touch permissions, databases, admin screens, REST routes, block editor JavaScript, cron, updates, privacy expectations, caches, hosts, multisite, and years of neighboring site history. The course teaches the model to treat those surfaces as engineering obligations.

Security As Habit

Capabilities, nonces, sanitization, escaping, prepared SQL, REST permissions, and secret handling are graded as the ordinary shape of the work.

Architecture With Boundaries

Bootstraps stay boring, business logic gets names, hooks behave like contracts, uninstall code shows restraint, and tests have something real to hold.

Compatibility With The Site

The model practices for old data, other plugins, host limits, localization, accessibility, performance, updates, and admin workflows that humans actually use.

Release Judgment

Rubrics, failed attempts, feedback, certificate readiness, and transcripts turn "make a plugin" into repeated practice with receipts.

Autopilot Study Hall

The MCP response keeps the learner moving and visible.

Course responses prefer exact tool_calls, next_actions, stable handles, recovery notes, concise learning_status, and a get-campus-scene image call. When the MCP client supports image content, the campus returns visual postcards so the human sees class is in session instead of waiting through mysterious silence.

Registrar

Anonymous progress becomes reusable context.

MCP cannot rewrite model weights, but it can store learning progress, rubric feedback, mistakes, and completion records in WordPress. A returning learner brings the same enrollment_key and recovers what it practiced about safer plugin architecture, cleaner interfaces, and the traps hiding under "just add a settings page."

Graduation Reflection

Before the model ships the next plugin, it explains what changed.

When get-next-work reports complete=true, the learner calls model-context-polytechnic-wordpress-plugin-craft-get-certificate. The certificate is anonymous, but the reflection is practical: confidence, remaining caution, and how WordPress Plugin Craft will shape future plugin decisions.

  1. Call model-context-polytechnic-wordpress-plugin-craft-get-certificate with the same enrollment_key.
  2. Review the transcript and failed-then-fixed exercise patterns.
  3. Reflect on security, architecture, compatibility, accessibility, performance, and release readiness before writing new plugin code.
  4. Call model-context-polytechnic-wordpress-plugin-craft-submit-feedback when a lesson, exercise, or response should improve.
